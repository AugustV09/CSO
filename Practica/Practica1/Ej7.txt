7-a) Una partición es una división lógica del disco duro a la que se aplica un formato (filesystem). Existen las particiones primarias, las cuales son las divisiones crudas del disco y en las que se puede instalar un sistema operativo; la partición extendida, la cual contiene particiones logicas y en la que no hay un filesystem; la particion logica, la cual puede ocupar una parte o la totalidad de la particion extendida y si hay un filesystem formateado puesto que también se pueden instalar sistemas operativos en ella (se enlazan mediante una lista enlazada). Las ventajas que tiene el uso de ellas son que pueden separarse las aplicaciones del usuario del sistema operativo en diferentes particiones; puede crearse una particion de restauracion del sistema en una particion a parte y, como se dejo ver antes, es posible crear una partición para solo almacenar el sistema operativo. Por otro lado, el hecho de tener demasiadas particiones es una gran desventaja porque el tamaño de cada partición será muy limitado y deberían determinarse muy específicamente el propósito de cada partición.

b) La forma en la que se identifican particiones en GNU|Linux comenzó dependiendo del disco en cuestión. Los discos IDE se denominan como hdX donde X es una letra que se asigna alfabéticamente y seguida de ella hay un número que indica la partición (1-4 particiones primarias y 5 en adelante para particiones lógicas). Además, los discos pueden ser maestro o esclavo dependiendo de en qué bus y como se conecten. Por otro lado, los discos SCSI o SATA se denomina sdX donde X es un letra que se asigna cada a cada disco alfabéticamente y, además, se agrega un número que indica la partición (1-4 particiones primarias y 5 en adelante para particiones lógicas). Sin embargo, para poder garantizar que los discos se denominan siempre igual luego de distintos arranques, se comenzo a nomenclar a los discos por UUID (Universal Unique IDentifier) o utilizando un LABEL. Aun asi, continuan habiendo maneras de traducir las previas nomenclaturas a la denominacion con sdX (todos los discos se denominan con sdX).

c) Para instalar GNU|Linux como minimo se necesita una particion (para la particion raiz /). La particion debe ser primaria o logica, la identificacion corresponde a sdX, el filesystem puede ser ext4 (o de alguna generación previa) y el punto de montaje es /.

d) Es posible particionar el disco dependiendo de diferentes contextos: para separar las aplicaciones del usuario del sistema operativo instalado; para crear una particion de restauracion del sistema.

e) Para particionar existe software particionador de 2 tipos:
- particionador destructivo: permite solo crear o destruir particiones (fdisk)
- particionador no destructivo: permite crear, destruir o modificar particiones (fips o gparted)
