1-a) Las caracteriticas mas importantes de GNU/Linux son que es codigo abierto, por lo cual cualquier persona puede ver, estudiar su contenido y hasta personalizarlo; es gratuito y de libre distribucion, por lo cual es posible descargarla desde la web y no se cobrara ningun servicio por su uso; es UNIX-like, por lo cual es compatible con el sistema operativo UNIX; es multiusuario, multiprocesador y multitarea; posee diversos interpretes de comandos para que se comunique el usuario con el SO; es latmente portable y trata a todo como un archivo.
b) Es posible comparar a GNU|Linux con Windows en que el primero es un software libre, de costo nulo (tambien es de libre distribucion), que se distribuye con su codigo fuente y que puede estudiarse, copiarse y modificarse y hasta que se corrige mas rapido puesto que se encuentra a cargo de los voluntarios; mientras que Windows es un software propietario, que tiene un costo asociado, no se distribuye libremente y menos lo hace con su codigo fuente y la corrección antes fallas es solucionado por el mismo propietario.
c) GNU, hoy dia, es todo aquel software que actua como las librerias y utilidades de los sistemas operativos UNIX-like (tipo UNIX).
d) El proyecto GNU comenzó como un intento de sistema operativo que contaba con programas compatibles con UNIX, de los cuales algunos resultaron ser mejores que los programas originales en UNIX, pero que no logró definir un kernel firme y estable para que este haga uso de las utilidades que ya ofrecía el proyecto. Tuvo intentos de adquirir un kernel pero no resultaron satisfactorios hasta que se combinó con el kernel Linux.
e) La multitarea se refiere a la capacidad de hacer multiples tareas en simultaneo. Los sistemas operativos la implementan, permitiendo que diversos procesos se ejecuten en simultáneo en la CPU al dividir su tiempo disponible entre los diferentes procesos. GNU/Linux hace uso de ella.
f) POSIX (Portable Operating System Interface) es una serie de normas, estándares y convenciones creadas por la comunidad de usuarios de Unix. Su objetivo es mejorar la compatibilidad entre diferentes sistemas operativos de tipo Unix y proporcionar una base para que las aplicaciones sean más portables entre estos sistemas operativos al definir una interfaz estándar para el sistema operativo (esto incluye el intérprete de comandos y las utilidades).
